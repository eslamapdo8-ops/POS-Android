# نظام نقاط البيع - متجر ملابس (POS MVP)

تطبيق نقاط بيع مبسّط لمحل ملابس، مبني بـ Kotlin و Room Database.

## المميزات

- **شاشة البيع**: شبكة منتجات مع بحث ومسح باركود وسلة مشتريات
- **إدارة المنتجات**: إضافة وتعديل وحذف المنتجات
- **مسح الباركود**: عبر كاميرا الجهاز (ML Kit + CameraX)
- **الطباعة**: دعم طابعات الحرارية عبر البلوتوث (ESC/POS)
- **تقرير المبيعات**: عرض مبيعات اليوم والإجمالي
- **قاعدة بيانات محلية**: Room Database (المصدر الوحيد للحقيقة)
- **واجهة عربية**: RTL بالكامل

## المتطلبات

- Android Studio (أحدث إصدار)
- JDK 17
- minSdk 24 / targetSdk 34

## البناء

```bash
./gradlew assembleDebug
```

الملف الناتج: `app/build/outputs/apk/debug/app-debug.apk`

## هيكل المشروع

```
app/src/main/java/com/pos/clothingstore/
├── POSApplication.kt          # فئة التطبيق + بيانات تجريبية
├── data/
│   ├── AppDatabase.kt         # قاعدة بيانات Room
│   ├── entity/                # الكيانات (Product, Sale, SaleItem, User)
│   └── dao/                   # واجهات الوصول للبيانات
├── ui/
│   ├── MainActivity.kt        # النشاط الرئيسي + القائمة الجانبية
│   ├── pos/                   # شاشة البيع
│   ├── products/              # إدارة المنتجات
│   ├── reports/               # تقرير المبيعات
│   └── scanner/               # مسح الباركود
└── printer/                   # طباعة البلوتوث
```

## قاعدة البيانات (4 جداول)

| الجدول | الوصف |
|--------|-------|
| products | المنتجات (باركود، اسم، سعر، كمية، مورد) |
| sales | فواتير البيع (تاريخ، إجمالي، طريقة دفع) |
| sale_items | بنود الفاتورة (مرتبطة بـ sales) |
| users | المستخدمون (بريد، دور) |

## CI/CD

يتم بناء APK تلقائياً عند الدفع عبر GitHub Actions.
يتم اختبار APK على محاكي Android API 26.
